Homologation

Pour exécuter les tests il faut suivre les descriptions données dans la feuille "Plans de test" colonne H "Cas de test".

Dans le cadre de l'homologation (test 0.1-060 excepté), il faut lancer le serveur DMPCProxy avec la configuration Spring, dmp-devHomologation.xml fournie :

%> java -cp dmpc-proxy-ws.jar fr.devcoop.dmpc.proxy.ws.server.DMPCProxyServer  / 
        -proxyURL http://127.0.0.1:9999/dmpcProxy -springContext META-INF/spring/dmpc-homologation.xml

Si l'erreur suivante apparait lors du démarrage du serveur :

Test 0.1-060

Dans ce test, il faut lancer plusieurs fois un même test d'existence sur un DMP avec des configurations du DMPC différents. La ligne de commande pour lancer les DMPCProxy est de la forme : 

%> java -cp dmpc-proxy-ws.jar fr.devcoop.dmpc.proxy.ws.server.DMPCProxyServer  / 
        -proxyURL http://127.0.0.1:9999/dmpcProxy -springContext META-INF/spring/CONFIGURATION-SPRING-A-CHANGER.xml

Il suffit de changer CONFIGURATION-SPRING-A-CHANGER.xml.

  1. Test dans "Environnement d'homologation", le serveur proxy est lancé avec pour config : dmpc-homologation.xml

    Test passant.

  2. Test dans "Serveur Bac à sable (dev1)", le serveur proxy est lancé avec pour config : dmpc-dev1.xml

    Test passant.

  3. Test dans "Environnement Atos", le serveur proxy est lancé avec pour config : dmpc-atos.xml

    Ce test est non passant et génère une exception dont le message est : 

    Exception in thread "main" fr.devcoop.dmpc.proxy.DMPCProxyException: com.ctc.wstx.exc.WstxIOException: 
sun.security.validator.ValidatorException: PKIX path building failed: 
sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target
  4. Test dans "Environnement Fake", le serveur proxy est lancé avec pour config : dmpc-fake.xml

    Ce test est non passant et génère une exception dont le message est : 

    Exception in thread "main" fr.devcoop.dmpc.proxy.DMPCProxyException: com.ctc.wstx.exc.WstxIOException: 
The https URL hostname does not match the Common Name (CN) on the server certificate.  
To disable this check (NOT recommended for production) set the CXF client TLS configuration property "disableCNCheck" to true.
