DMPC Proxy WS

Introduction

Ce document a pour but de montrer simplement l'ensemble des fonctionnalités du serveur proxy DMP, concernant les transactions spécifiées dans le DSFT du DMP.

Architecture

L'architecture du proxy WS peut être représentée par ce schéma :

Prérequis

Un JDK 8.0 installé. Pour vérifier la version :

  • Runtime 
    $> java -version

    Résultat attendu :

    java version "1.8.0_131"
Java(TM) SE Runtime Environment (build 1.8.0_131-b11)
Java HotSpot(TM) 64-Bit Server VM (build 25.131-b11, mixed mode)
  • Compilateur 
    $> javac -version

    Résultat attendu :

    javac 1.8.0_131

    Si problème, il faut configurer la variable d'envrironnement JAVA_HOME. Exemple :

    JAVA_HOME=C:\Program Files\Java\jdk1.8.0

Côté SERVEUR

Démarrage du serveur Proxy

le proxy WS est un programme java qui reçoit les requetes simples et les retransmet au DMP dans le bon formalisme et dans le respect des différentes transactions, il interprète les réponses du serveur DMP et les retourne au client initial dans un formalisme simplifié.

Pour démarrer le serveur, depuis une ligne de commande exécuter : 

$> java -Dspring.config.location=application.yml -jar dmpc-proxy-ws.jar

Au lancement du serveur est chargé un fichier de configuration application.yml permettant de charger différents KeyStore les informations concernant le LPS et le serveur DMP. Il est initialisé par défaut sur un environnement de test. Néanmoins pour plus d'infos : ./configurationServeurProxy.html

Une fois démarré, le serveur est accessible à l'url suivante : http://127.0.0.1:9999/dmpcProxy

Pour obtenir, le schéma wsdl, il suffit de suffixer l'url par "?wsdl" : http://127.0.0.1:9999/dmpcProxy?wsdl.

Côté CLIENT

Exemple de code client en JAVA

Le code ci dessous est donnée à titre d'exemple de client en Java utilisant CXF, il montre l'usage possible des différents web services fournis. 

// Création du client WS 
JaxWsProxyFactoryBean factory = new JaxWsProxyFactoryBean();
factory.getInInterceptors().add(new LoggingInInterceptor());
factory.getOutInterceptors().add(new LoggingOutInterceptor());
factory.setServiceClass(DMPCProxy.class);
factory.setAddress("http://127.0.0.1:9999/dmpcProxy");
DMPCProxy proxyService = (DMPCProxy) factory.create();

// Création du contexte qui sera utilisé tout le long de la conversation avec le proxy.
DMPCContext context = new DMPCContext("user", "password");         
DMPCAuthor author = new DMPCAuthor("internalId", "Durand", "Pierre");
context.setAuthor(author);
author.setRole(proxyService.getCodesFor(ASIPJeuxValeurs.SUBJECT_ROLE).get(3).getCode()); // Médecin       
author.setSpecialite(proxyService.getCodesFor(ASIPJeuxValeurs.AUTHOR_SPECIALITY).get(33).getCode()); // G15_10/SM30 Néphrologie 
// secteurActivite est obtenu aupres de l'annuaire gip-cps plus d'infos sur documentation configurationServeurProxy
author.setSecteurActivite(proxyService.getCodesFor(ASIPJeuxValeurs.HEALTH_CARE_FACILITY_TYPE_CODE).get(40).getCode());  // ici c'est SA41

Profil Gestion Administrative Patient

La gestion administrative du patient dans le DMP est constituéé de différentes transactions. La DevBox-Santé DMP met en oeuvre les transactions suivantes :

  • TD0.2 : Test d'existence d'un DMP et vérification de l'autorisation d'accès.
  • TD0.3 : Mise à jour de l'autorisation d'accès.
  • TD1.1 : Création d'un DMP
  • TD1.5 : Accès internet du patient.

Pour plus de renseignement sur la description des différentes transactions, se référer au DSFT.

Exemple de scénario

Un scénario classique de création d'un DMP consistera à vérifier l'existence du DMP d'un patient, le créer si besoin puis générer les Accès internet du patient. Si le DMP existe déjà, il faut vérifier l'autorisation du PS à accéder au DMP du patient.

Ce qui à partir d'un client java du Proxy WS donnera :

final String insc = "1164485058822081751070"; // INSC existant  dans le DMP

////////////////////////////////////////////////////////////////////////
// TD0.2 : Test d'existence
////////////////////////////////////////////////////////////////////////        
// Test d'existence de l'insc !
TD02Request td02request = new TD02Request(insc);
TD02Response response = proxyService.td02Exist(context, td02request);
DMPCPatient patient = response.getPatient();
DMPCRole role = response.getRole();

if (patient == null) {
    ////////////////////////////////////////////////////////////////////
    // TD 1.1 : Création du Patient à partir des données connues du SI
    ////////////////////////////////////////////////////////////////////
    TD11Request request = new TD11Request();
    //Création des informations patients
    patient = new DMPCPatient();
    patient.setInsc(insc);
    patient.setNom("Lascar");
    patient.setPrenom("Germain");
    patient.setSexe(DMPCSexe.M);
    patient.setTelephone("0987654321");
    patient.setDateDeNaissance("20131212");
    request.setPatient(patient);
    // Création des informations lu de la carte vitale
    DMPCCarteVitaleInfo carteVitaleInfo = new DMPCCarteVitaleInfo();
    carteVitaleInfo.setDateNaissance("131212");
    carteVitaleInfo.setNom("Lascar");
    carteVitaleInfo.setPrenom("Germain");
    request.setCarteVitaleInfo(carteVitaleInfo);
    // Récupération des informations de consentement du patient
    request.setConsentementOuverture(true);
    request.setOppositionBrisDeGlace(false);
    request.setOppositionUrgence(false);
    // Exécution de la requête de création du DMP
    final TD11Response td11Response = proxyService.td11CreationDmp(context, request);

    // Récupération du patient donné dans la réponse.
    patient = td11Response.getPatient();

    ////////////////////////////////////////////////////////////////////
    // TD 1.5a : Création de l'accès internet du patient
    ////////////////////////////////////////////////////////////////////
    //Création des informations OTP (One Time Password) du patient
    TD15aRequest td15aRequest = new TD15aRequest();
    td15aRequest.setPatient(patient);
    td15aRequest.setTelephone("0987654321");
    // Exécution de la requête de création d'accès internet
    final TD15aResponse td15aResponse = proxyService.td15aCreerAccesInternetPatient(context, td15aRequest);
    // Récupération de la réponse, notamment le fichier PDF binaire à imprimer au patient.
    sendToPrinter(td15aResponse.getFormulairePatient());
} else {
    // A t'on une authorisation sur DMP  
    if (role == null) {
        // sinon on en demande une
        // sinon on en demande une

        ////////////////////////////////////////////////////////////////////
        // TD 0.3 : Gestion des autorisations
        ////////////////////////////////////////////////////////////////////
        TD03Request td03Request = new TD03Request(insc, DMPCRole.STANDARD);

        ////////////////////////////////////////////////////////////////////
        // TD 0.3 : Gestion des autorisations
        ////////////////////////////////////////////////////////////////////
        TD03Request td03Request = new TD03Request(insc, DMPCRole.STANDARD);
        proxyService.td03AddAuthorization(context, td03Request);
    }
}
